# Техническое задание

## Курсовой проект

### «Сервис учёта успеваемости студентов»

---

# 1. Общая информация о проекте

## Название проекта

Сервис учёта успеваемости студентов.

## Цель проекта

Разработать современный web‑сервис для автоматизации учебного процесса учреждения образования. Система должна обеспечивать:

* учёт успеваемости студентов;
* хранение оценок и рейтингов;
* управление расписанием;
* контроль посещаемости;
* взаимодействие студентов и преподавателей;
* обработку заявок и обращений;
* разграничение ролей пользователей;
* формирование статистики и отчётов.

Проект разрабатывается как web‑приложение с клиент‑серверной архитектурой.

# 3. Общая идея проекта

Система представляет собой единый портал учебного учреждения.

Каждый пользователь входит в систему под своей ролью и получает доступ только к нужному функционалу.

Сервис должен поддерживать:

* личные кабинеты;
* электронный журнал;
* систему рейтинга;
* расписание занятий;
* контроль посещаемости;
* уведомления;
* загрузку файлов;
* систему заявок, заказа справок;
* аналитику и статистику.
* новости
* контакты
* техподдержка
* зачётка

# 5. Основной функционал системы

# 5.1 Авторизация и аутентификация

Система должна поддерживать:

* регистрацию пользователей;
* вход по логину и паролю;
* JWT авторизацию;
* восстановление пароля;
* смену пароля;
* разграничение прав доступа.

Дополнительно:

* подтверждение email;
* защита от несанкционированного доступа;
* ограничение доступа по ролям.

---

# 5.2 Электронный журнал

Функции:

* выставление оценок;
* редактирование оценок;
* просмотр истории изменений;
* расчёт среднего балла;
* фильтрация по предметам;
* фильтрация по семестрам.

Типы оценок:

* лабораторная работа;
* практическая работа;
* контрольная;
* экзамен;
* зачёт;
* домашнее задание.

---

# 5.3 Система рейтинга

Система должна автоматически рассчитывать:

* средний балл;
* рейтинг внутри группы;
* рейтинг потока;
* процент посещаемости;
* активность студента.

Формула рейтинга:

Рейтинг =

* 70% оценки
* 20% посещаемость
* 10% активность

Дополнительно:

* таблица лидеров;
* лучшие студенты месяца;
* аналитика прогресса.

---

# 5.4 Учёт посещаемости

Преподаватель должен иметь возможность:

* отмечать присутствие;
* отмечать опоздания;
* отмечать уважительные пропуски;
* добавлять комментарии.

Система должна:

* подсчитывать количество пропусков;
* показывать статистику;
* отправлять предупреждения.

---

# 5.5 Расписание занятий

Функции:

* просмотр расписания;
* отображение аудиторий;
* отображение преподавателей;
* отображение времени занятий;
* фильтрация по группе.

Дополнительно:

* календарный режим;
* уведомления об изменениях.

---

# 5.6 Домашние задания

Преподаватель:

* создаёт задания;
* указывает дедлайн;
* прикрепляет файлы.

Студент:

* загружает решение;
* отслеживает статус проверки.

Система:

* уведомляет о дедлайнах;
* показывает просроченные задания.

---

# 5.7 Система заявок

Типы заявок:

* справка об обучении;
* перенос занятия;
* академическая справка;
* исправление оценки;
* пропуск по уважительной причине;
* техническая проблема.

Статусы:

* создана;
* в обработке;
* одобрена;
* отклонена.

---

# 5.8 Уведомления

Система уведомлений должна поддерживать:

* email уведомления;
* внутренние уведомления;
* уведомления о новых оценках;
* уведомления об изменении расписания;
* уведомления о дедлайнах.
Вообщем можно сделать отдельную вкладку для уведомлений и там их просматривать. 

---

# 5.9 Аналитика и статистика

Система должна строить:

* графики успеваемости;
* графики посещаемости;
* сравнение студентов;
* отчёты по дисциплинам;
* отчёты по преподавателям.

Библиотеки:

* Chart.js
* matplotlib (если понадобится серверная генерация)

---

# 5.10 Чат и объявления

Функции:

* объявления преподавателей;
* новости группы;
* комментарии;
* чат преподаватель ↔ студент.

Для чата:

* Django Channels
